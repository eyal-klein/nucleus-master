## Documentation Standards

### Markdown Format
All documentation uses **GitHub-flavored Markdown** with:
- Clear headings and structure
- Tables for organized information
- Code blocks with syntax highlighting
- Inline links for navigation

### Version Control
- All docs tracked in Git
- Updated with each major release
- Historical versions in `archive/`

### Language Support
- **English:** Primary language for technical docs
- **Hebrew:** Client-facing and philosophical content
- **Bilingual:** Manifesto available in both languages

## Contributing to Documentation

### Adding New Documentation
1. Place in appropriate directory
2. Update this index
3. Link from relevant docs
4. Follow Markdown standards

### Updating Existing Documentation
1. Update the "Last Updated" date
2. Increment version if major changes
3. Move old version to `archive/` if needed
